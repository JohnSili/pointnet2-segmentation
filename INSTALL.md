# Инструкция по установке и запуску

## Шаг 1: Настройка окружения

Запустите скрипт настройки:
```bash
./setup.sh
```

Или вручную:
```bash
# Создать виртуальное окружение
python3 -m venv venv

# Активировать окружение
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

**Примечание:** Установка PyTorch может занять несколько минут (размер ~900MB).

## Шаг 2: Проверка данных

Ваши данные находятся в:
```
3011-20251217T195928Z-1-001/3011/
```

Формат: PLY файлы с координатами (x, y, z) и метками классов.

## Шаг 3: Быстрый запуск обучения

После завершения установки:

```bash
# Активировать окружение
source venv/bin/activate

# Быстрый старт
./quick_start.sh
```

Или вручную:
```bash
python train.py \
    --data_dir 3011-20251217T195928Z-1-001 \
    --area 3011 \
    --num_points 2048 \
    --batch_size 4 \
    --epochs 50 \
    --num_classes 13
```

## Шаг 4: Мониторинг обучения

В другом терминале:
```bash
source venv/bin/activate
tensorboard --logdir ./logs
```

Откройте браузер: http://localhost:6006

## Шаг 5: Тестирование модели

После обучения:
```bash
python test.py \
    --data_dir 3011-20251217T195928Z-1-001 \
    --area 3011 \
    --checkpoint ./checkpoints/best_model.pth \
    --num_classes 13
```

## Шаг 6: Визуализация

```bash
python visualize.py \
    --data_dir 3011-20251217T195928Z-1-001 \
    --area 3011 \
    --checkpoint ./checkpoints/best_model.pth \
    --num_samples 5 \
    --output_dir ./visualizations
```

## Параметры для ваших данных

- **Количество классов:** Проверьте уникальные метки в данных (обычно 0-12)
- **Размер батча:** Начните с 4, увеличьте если есть память GPU
- **Количество точек:** 2048 или 4096 (зависит от размера ваших облаков)

## Устранение проблем

### Проблема: Out of Memory
- Уменьшите `--batch_size` до 2
- Уменьшите `--num_points` до 1024

### Проблема: Медленное обучение
- Используйте GPU: `--device cuda`
- Уменьшите `--num_points`

### Проблема: Данные не загружаются
- Проверьте путь: `--data_dir 3011-20251217T195928Z-1-001`
- Проверьте область: `--area 3011`
